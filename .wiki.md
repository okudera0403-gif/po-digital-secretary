# Project Summary
The "PO Digital Secretary" is a SaaS web application designed specifically for prosthetics and orthotics clinics. It streamlines patient case management and automates document generation, significantly enhancing operational efficiency for small teams. The platform captures essential patient data, generates manufacturing instructions, and provides cost estimates, thereby improving service quality and patient outcomes.

# Project Module Description
- **Case Input Form**: Form for entering patient information, including remarks, photos, and videos.
- **Case List Screen**: Displays patient cases with options for editing and document generation.
- **Case Detail Screen**: Mobile-optimized view with tabs for Basic Info, Estimate, Manufacturing Instruction, and AI Opinion Draft.
- **Manufacturing Instruction Screen**: Generates printable PDF instructions based on selected products.
- **Estimate Generation**: Produces cost estimates for selected products and components.
- **AI Medical Opinion Draft**: Utilizes AI to create formatted medical justification texts for official forms.
- **Parts Search**: Allows searching for parts categorized by product type.
- **Product Management**: Manages product categories, products, and manufacturing elements, integrating data from external sources.
- **Migration Management**: Updates the database schema to align with changes in the ORM model.
- **Data Importer**: Imports lower limb orthotic device data from Excel, updating existing database tables.
- **Quick Entry Page**: Combines workflows for case entry, manufacturing instructions, estimate generation, and opinion drafting.
- **AI Opinion Page**: Dedicated page for generating AI-based medical opinions.
- **Estimate System**: Redesigned for effective management and calculation of estimates.
- **Estimate Page**: Updated feature for generating estimates based on a new data structure.
- **Draft Management**: New module for managing draft cases, including saving, finalizing, and deleting drafts.

# File Description Inventory
- **app/backend**: Contains backend services and APIs for case management, user authentication, and database interactions.
- **app/frontend**: The frontend application providing user interfaces for case management and document generation.
- **README.md**: Documentation for both backend and frontend modules.
- **requirements.txt**: Python dependencies for the backend.
- **package.json**: JavaScript dependencies for the frontend.
- **app/backend/data_models/**: JSON files defining structures for case data storage, product categories, and products.
- **app/backend/models/**: Model definitions for cases, products, and related entities.
- **app/backend/routers/**: Manages API routes related to case operations, products, and manufacturing elements, including drafts.
- **app/frontend/src/**: Contains the main application components and pages, including updates for handling new product data and AI opinion generation.
- **app/backend/mock_data/**: JSON files for testing with mock data for finished parts, manufacturing elements, product categories, and products.

# Technology Stack
- **Backend**: Python, FastAPI, SQLAlchemy, Alembic
- **Frontend**: React, TypeScript, Vite, Tailwind CSS, jsPDF
- **Database**: PostgreSQL

# Usage
1. **Install Dependencies**:
   - For the backend: Navigate to `app/backend` and run `pip install -r requirements.txt`.
   - For the frontend: Navigate to `app/frontend` and run `pnpm install`.

2. **Build the Application**:
   - For the backend: Use the command `uvicorn main:app --reload` to start the backend server.
   - For the frontend: Run `pnpm run build` to compile the frontend application.

3. **Run the Application**:
   - Start the backend server and then the frontend server to access the application.
